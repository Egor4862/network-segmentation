# Сегментация корпоративной сети автосалона Legend
## Задача
Дана корпоративная сеть автосалона без деления на сегменты. Все сервисы находятся в одной плоской сети. Нужно разделить их на сегменты с учётом принципов информационной безопасности: минимизировать зону поражения при компрометации одного из сегментов и разграничить доступ между отделами.
## Сегменты сети
- **DMZ**: сюда выносится то, к чему нужен доступ снаружи (интернет-портал, дилерский портал, СКУД) вместе с их базами. Из DMZ во внутреннюю сеть доступа нет.
- **Access-зона**: VPN и Terminal Service Gateway ведут не сразу на рабочие станции, а через Jump-сервер. Так снаружи не видно сразу всю сеть.
- **Application-сегмент**: обычные внутренние системы: CRM, кадры, бухгалтерия, юротдел, телефония.
- **Mechanics-сегмент**: система диагностики авто и система коллерной выносятся отдельно, доступ только у слесарей.
- **Finance**: банк-клиент изолируется отдельно, доступ только с рабочих мест бухгалтерии.
- **AD**: Active Directory и почта выносятся отдельно, остальные сегменты обращаются туда только за авторизацией.
- **Users**: рабочие станции бухгалтерии и ноутбуки менеджеров разделены, чтобы отделы не имели доступа друг к другу.
- **mgmt**: админ и терминальный сервер с полным доступом изолированы от обычных пользователей.

## Как можно усилить безопасность
- Разбить DMZ на два уровня, чтобы базы данных публичных сервисов были отдельно от самих сервисов
- Прописать конкретные правила firewall между сегментами, а не только общую логику
- Разделить слесарей и менеджеров ещё строже, если появятся новые отделы

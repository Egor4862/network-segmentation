# Сегментация корпоративной сети автосалона Legend

## Задача
Дана корпоративная сеть автосалона без деления на сегменты — все сервисы находятся в одной плоской сети. Нужно разделить их на логические сегменты с учётом принципов информационной безопасности: минимизировать зону поражения при компрометации одного из сегментов и разграничить доступ между отделами.

## Как разделил

- **DMZ** — сюда вынес то, к чему нужен доступ снаружи (интернет-портал, дилерский портал, СКУД) вместе с их базами. Из DMZ во внутреннюю сеть доступа нет.
- **Access-зона** — VPN и Terminal Service Gateway ведут не сразу на рабочие станции, а через Jump-сервер. Так снаружи не видно сразу всю сеть.
- **Application-сегмент** — обычные внутренние системы: CRM, кадры, бухгалтерия, юротдел, телефония.
- **Mechanics-сегмент** — систему диагностики авто и систему коллерной вынес отдельно, доступ только у слесарей. Остальным она не нужна.
- **Finance** — банк-клиент изолировал отдельно, доступ только с рабочих мест бухгалтерии.
- **AD** — Active Directory и почта отдельно, остальные сегменты ходят туда только за авторизацией.
- **Users** — рабочие станции бухгалтерии и ноутбуки менеджеров разделены, чтобы отделы не лазили друг к другу.
- **mgmt** — админ и терминальный сервер с полным доступом, изолированы от обычных пользователей.

## Что можно доделать
- Разбить DMZ на два уровня, чтобы базы данных публичных сервисов были отдельно от самих сервисов
- Прописать конкретные правила firewall между сегментами, а не только общую логику
- Разделить слесарей и менеджеров ещё строже, если появятся новые отделы
